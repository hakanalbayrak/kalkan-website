# Kalkan Homepage Plan (V1)

This document defines the first production-ready homepage structure and copy direction for `kalkan.website`, built for WordPress + Blocksy.

## 1. Recommended Homepage Section Order

1. Hero
2. How It Works
3. Core Features
4. Communication Reporting
5. Trust and Privacy
6. App Store CTA
7. FAQ Preview

## 2. Headline and Subheadline Options

## Hero Option A
- Headline: Know who is calling. Stay safer every day.
- Subheadline: Kalkan helps you identify unknown callers, reduce spam interruptions, and report suspicious calls in seconds.

## Hero Option B
- Headline: Fewer spam calls, more peace of mind.
- Subheadline: Use Kalkan to identify calls, block unwanted numbers, and help protect others through simple reporting.

## Hero Option C
- Headline: Caller safety made simple.
- Subheadline: Kalkan gives clear caller information and easy protection tools for you and your family.

## 3. Recommended Copy for Each Section

## Section 1: Hero
- Suggested heading: Caller safety made simple.
- Suggested copy block: Kalkan helps you understand unknown calls before you answer. Identify suspicious numbers, reduce spam disruptions, and protect daily communication with a simple, easy-to-use app.
- Suggested supporting line: Built for everyday use, including families and older adults.

## Section 2: How It Works
- Suggested heading: How Kalkan works
- Suggested copy block intro: Start in minutes with three simple steps.
- Suggested steps:
1. Download Kalkan from the App Store.
2. Turn on spam protection and caller identification.
3. Report suspicious calls when needed to help improve community safety signals.

## Section 3: Core Features
- Suggested heading: Core features that protect your calls
- Suggested copy block intro: Kalkan focuses on practical protection, not complexity.
- Suggested feature cards:
1. Spam Protection
- Copy: Reduce interruptions from unwanted callers.
2. Caller Identification
- Copy: See clearer caller context for unknown numbers.
3. Extra Protection
- Copy: Add stronger day-to-day call safety with simple controls.

## Section 4: Communication Reporting
- Suggested heading: Report suspicious calls in seconds
- Suggested copy block: If a call looks suspicious, you can report it directly in the app. Communication Reporting is free. Your reports help improve community awareness and support safer calling for everyone.
- Suggested supporting line: No technical setup needed.

## Section 5: Trust and Privacy
- Suggested heading: Built with trust and privacy in mind
- Suggested copy block: Kalkan is designed to be clear, respectful, and easy to understand. Privacy and transparent communication are core priorities, with clear policy pages and support access on the website.
- Suggested trust links text: Review our Privacy Policy, Terms, and Contact/Support pages.

## Section 6: App Store CTA
- Suggested heading: Get Kalkan on the App Store
- Suggested copy block: Ready for safer calls and clearer caller information? Install Kalkan and set up protection in minutes.
- Suggested supporting line: Simple onboarding for all age groups.

## Section 7: FAQ Preview
- Suggested heading: Common questions
- Suggested copy block: Quick answers before you get started.
- Suggested note: Show 4-6 short questions on homepage and link to full FAQ page.

## 6. Implementation Notes for WordPress + Blocksy

- Use a standard WordPress page for Home and build sections with Gutenberg core blocks.
- Keep one column flow on mobile first; expand to two/three-column layouts only where needed on larger screens.
- Recommended blocks by section:
1. Hero: Group + Heading + Paragraph + Buttons
2. How It Works: Group + Heading + Numbered List
3. Core Features: Group + Heading + Columns (3 feature cards)
4. Communication Reporting: Group + Heading + Paragraph + Button
5. Trust and Privacy: Group + Heading + Paragraph + Buttons/Links
6. App Store CTA: Group + Heading + Paragraph + Button
7. FAQ Preview: Group + Heading + List or Details blocks + link button
- Keep copy short: 1 heading + 1 short paragraph + clear CTA per section.
- Avoid custom frameworks, custom build tooling, and heavy JavaScript.
- Use Blocksy typography and spacing settings; do not over-customize in v1.
- Validate legibility on mobile (large tap targets, readable font sizes, clear spacing).
- Keep links to Privacy Policy, Terms, and Contact/Support visible in both homepage body and footer.
